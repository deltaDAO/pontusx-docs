# The EU Data Act 

## Data Spaces: Architecture of Sovereignty 
Implementing the requirements of the Data Act requires a new approach to data sharing. Data Spaces differ fundamentally from traditional cloud platforms or data lakes. 

### The Federation Principle
In a Data Space, data is not stored centrally. It remains with the owner (‘Data Owner’). Exchange takes place peer-to-peer, based on defined rules. This reflects the decentralised nature of the European economy and prevents the emergence of monolithic data monopolies. 



### Compute-to-Data (C2D): The solution to the trade secret dilemma 
Perhaps the most important technology in Pontus-X for SMEs is Compute-to-Data. It directly addresses the conflict between Article 4 (duty of access) and Article 8 (protection of trade secrets) of the Data Act. 

How it works: 

Instead of sending raw data, the data owner allows the data user to run an algorithm (e.g. an AI training script) in a secure environment (container) hosted by the data owner. 

    - The user sends the algorithm. 

    - The algorithm runs locally on the owner’s data. 

    - Only the result (e.g. a trained model or a statistical analysis) is sent back. 

Strategic advantage: The raw data never leaves the SME’s jurisdiction. This eliminates the risk of reverse engineering whilst ensuring compliance with the Data Act (enabling data use). C2D transforms data access from a risk into a controlled service.
